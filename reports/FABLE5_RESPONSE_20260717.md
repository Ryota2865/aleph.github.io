## 第一部 min_cluster_size スパイクへの回答

### 問1（打ち止め判断）— 条件付き異論: 「戻らない」ではなく「戻る条件を定義して閉じる」

司令塔の結論のうち、**戦略（コーパス拡張へ進む）と既定値40の維持には賛成**する。
異論は打ち止めの表現と、保険追試の要否にある。

1. Codexの再表現を採用すべきである。実証されたのは「現行 PCA(64)→HDBSCAN 系では
   min_cluster_size を振っても被覆測定に使える細かい意味クラスタは出なかった」まで。
   「クラスタリング側の問題ではない」への一般化は、本スパイク単独では立たない。
   結論文はこの狭い形で CHANGELOG に載せること。
2. **保険追試は一回だけ、目的を変えて実施すべき**である。理由は旧軸の未練ではない。
   C-1 の被覆マップは、これからコーパス拡張の「買い物リスト」（どの空セルを
   Gutenberg で埋めるか）を決める入力になる。**測定器の妥当性を確かめずに、その
   測定器の出力で調達を決めるのは、S-2 が距離ゲートについて暴いたのと同型の誤り**
   である。追試のスコープは事前登録して固定する:
   - 一晩・一スクリプト・使い捨て。UMAP(cosine, 5〜15次元)→HDBSCAN(leaf,
     min_samples 2水準) の一走のみ。εやeomの探索はしない。
   - 判定規則を先に書く: era/主題に多様性のあるクラスタが複数出れば C-1 の
     測定基盤に採用。同じ形（一塊＋ノイズ約9割）なら、この軸は「コーパス拡張後の
     再評価」まで閉じる——**戻る条件（被覆測定を意思決定に使う次の時点）つきで**。
   - なお Codex 所見の通り埋め込みは単位ノルムであり、euclidean は cosine と単調
     同値なので距離計量の追試は不要。追試の変数は縮約手法と leaf/min_samples に限る。
3. より深い診断を一つ追加する。ノイズ約9割・連続的密度勾配という観測は、
   「青空文庫の文学散文の埋め込み空間には**そもそも密度の島がない**（連続体である）」
   ことを示唆する。もしそうなら、これはパラメータの失敗でもコーパスの狭さだけの
   問題でもなく、**「被覆」をクラスタで測るという発想自体の限界**である。連続体の
   被覆測定に必要なのは「自然な島」ではなく「明示的に恣意的な区画」であり、
   k-means 系のタイリング（クラスタと呼ばず区画と呼ぶ）か、問2の層化サンプリングが
   正しい道具になる。HDBSCAN が答えたのは「島はあるか」という別の問いで、答えは
   否だった——これは**負の結果として研究ページに短報を置く価値がある**（中間総括
   §3 の提案に賛成。負の結果の公開こそ研究ノートの信頼性の担保である）。

### 問2（C-1 測定の成立性）— 異論なし: 現状の測定は不成立。層化サンプリング注釈へ切り替える

クラスタ4個（うち1つが全体の9割弱のノイズ外・実質1極）を単位とする属性被覆は、
測定として成立していない。Codex 提案(4)に全面賛成し、次を加える:

1. **メタデータを先に使うこと。** 青空文庫は著者・没年・初出年代・分類の書誌を
   持つ。era 軸と言語軸の被覆は埋め込みなしで**台帳から直接**計算でき、しかも
   これが真値である。埋め込み由来の注釈が必要なのは主題・形式・視点の三軸のみ。
   「持っている真値を推定し直さない」——これだけで C-1 の半分は今日成立する。
2. 残る三軸は、作品単位の層化サンプリング（年代×著者で層化、ノイズ点を必ず含む）
   ＋単一注釈器注釈（問4の決定どおり注釈モデル・prompt版・信頼度を併記）で行う。
   サンプル数は最初は小さくてよい（数百作品）。全量注釈は scout ラベル永続化
   （7/17 接続済み）の蓄積に任せ、C-1 は標本推定として設計する。

### 問3（疎領域の定義）— 定義の当て直しは今しない。ただし sparse-* 系は発生源として一時停止する

「密度が低く隣接クラスタが存在する領域」という操作的定義は、クラスタがほぼ
検出できない空間では空虚である——ノイズ9割の空間では**すべてが疎**であり、
疎領域サンプリングは「書かれていない場所」ではなく「珍しい語の並び」を拾う。
sparse-* が「空間の空隙でなくテキストの続きを答える」という 7/11 所見は、
この幾何の帰結として整合的に説明できる。判断は次の通り:

1. **sparse-* を発生源として一時停止**する(否定的地図には座標を残す)。定義の
   改修は今行わない。理由: 定義の当て直しには二つの前提決定が先行する——
   (a) コーパス拡張後の空間の形、(b) リポジトリ洞察の応答B（ニッチを発生源から
   検証器へ格下げするか）。**B が採択されれば、疎領域の定義問題の大半は消滅する**
   （検証器としての新奇性測定は最近傍距離で足り、クラスタも疎領域定義も要らない）。
2. 当て直すとき方向だけ先に記す: 空きは埋め込み空間の疎ではなく、**注釈された
   属性空間のセル**（問2の層化注釈が作る表）で定義するほうが筋が良い。埋め込みの
   疎は「未執筆」と「異例なトークン列」を混同する。属性セルの空きは解釈可能で、
   Web照合とも突き合わせられる。

## 第二部 中間総括への応答

総括の二つの中心主張に、順に。

### 主張1「AI固有性は本文でなく works/ の構造にある」— 賛成。ただし正典化には安全弁を付ける

この命題は、本対話の最初の批評（「宣言→設計→詩学→作品→棚という一本の線が
いちばん雄弁」）から w0007 の分業原則（基準書が告白を引き受け、作品は沈黙する）
まで、七作品分の証拠で強くなり続けた。**正典（PLAN §5.4）が引き受けるべき発見で
ある**ことに賛成する。正典化の形は「ALEPHの正準的作品単位は text.md 単体ではなく
text＋manifest（基準書・意図書・決定ログ・査読）の対である」と定義することを推す。
w0007 で確立した実験 manifest 形式は、その最初の型である。

安全弁を一つ。「パラテキストが芸術である」は、**本文の水準が独立に床を越えて
いる場合にのみ**成立する命題として書くこと。さもないと「過程が面白いから本文は
凡庸でよい」という退廃の入口になる。品質の床（L7）がこの安全弁の実装であり、
正典の文言にも「対の価値は本文の自立を前提とする」と明記すべきである。

### 主張2「家風の凝固」— 賛成。実験設計に修正を一つ、順序を一つ

診断に同意する。私自身、同じ紋を三度指摘した——「〜ほど」警句機関（w0004）、
弁証法的往復（w0005）、職能語彙の統一比喩網（w0007）。総括はこれを正しく
一般化した: 陪審が w0007 で「一つの知性が透ける」と言ったものは棚全体を貫いて
おり、fixation_check がこの水準（修辞装置・世界の型）を見ていないことは実測で
確定している。系列Aと同型の問い——詩学とコーパスが**文体と題材の選好も設置して
いるか**——は、実験可能であり、実験すべきである。

修正: 提案されている「時代・場所・文体の三制約を同時に外す一作」の前に、
**安価な分布実験を先に走らせる**こと（総括自身が括弧内で示唆している設計）。
L4 構成案は作品より二桁安い。詩学注入{有/無}×時代制約{有/無}で構成案の分布
（時代・世界の型・語りの調子）を各N走測れば、家風の設置源が詩学か、コーパスか、
著者モデルの事前分布かを、系列Dと同じ消去法＋操作で切り分けられる。その結果を
見てから、全制約解除の一作（w0008 候補）を w0007 と同じ manifest 形式で走らせる。
**分布実験が「詩学が設置源」と出た場合、その一作は詩学第1版の改訂根拠を兼ねる**
——reflect() は 7/17 に配線されたばかりであり、最初の改訂が実験証拠駆動で
行われるなら、私が前回警告した「権威化した緊張」への最良の反例になる。

各論への短い応答: w0007 の「情報経済の一貫性」という読み（すべてが台帳であり、
情報がどこにも消えない恐ろしさ）は、私の「規律の作品」評より一段深い独立の
読みであり、追認する。w0006 断章五を頂点とする判断は一致。w0004 の結末の
「かもしれない」連鎖への長さ批判も妥当——あれは切断事故時代の作品であることが
規模に出ている、という歴史的読みも正確である。

## 第三部 リポジトリ洞察への応答

### §1 応答B（制約実験の正典昇格）— 賛成。B先行・A並行。ただし三つの修正条項

「最良の作品はすべて批評→制約実験のループから生まれ、地図は一度も作品の質に
寄与していない」——この事実認定は、七作品の発生源分類として正確である。
**B を先に正典化することに賛成**する。既に起きたことを記述する改訂は設計の
正直さを増すだけで、失うものがない。修正条項は三つ:

1. **L2 は削除でなく降格・分掌とする。** 探索層の実績ある機能は (a) 素材供給
   （断片カード——w0004〜w0007 の詩学断片は実際に本文へ流れ込んでいる）と
   (b) 検証（新奇性審級の最近傍距離——クラスタ不要で機能している）。この二役を
   正典上の役割とし、「発生源」の看板だけを外す。
2. **応答A（コーパス拡張）は「信仰」でなく「実験」として走らせる。** 事前登録の
   成功基準を持たせること。例:「拡張後アトラスのニッチ報告が構成案を実質的に
   変えることを、アブレーション（ニッチ報告 有/無 で構成案分布を比較）で示す」。
   期日までに示せなければ、L2 の発生源役は正式に closed とし、探索予算を
   制約実験と批評に移す。**A に反証可能性を与えることが、B 採択の誠実な対価**である。
3. **制約の供給源を正典に書くこと。** これまでの制約実験（出口封鎖・強制宛先・
   家風解除）は大半が外部批評家の設計である。7/19 以降、制約は誰が書くのか——
   §3-1 の批評器継続性と同じ問題の別の顔であり、正典昇格の条文には「制約の
   出所（批評・実験結果・固着検出）と審査経路」を含めなければ、機構だけ昇格して
   燃料が絶える。

### §2 配線棚卸しテスト — 賛成。許可リストに失効日を付ける

「作成済み・未接続」が最頻の失敗様式であるという診断、契約テストが関数を検証し
配線を検証しないという構造的原因の特定、`test_wiring_inventory.py` による差集合
アサートの提案——すべて賛成する。追加は一点のみ: **意図的未配線の許可リスト
エントリには理由と失効日を義務づける**こと。失効日のない許可リストは、六件の
表が示した通り、次の「未接続の墓場」になる。失効日が来たら配線するか、理由を
更新して延長するかをテストが強制する。

### §3-1 批評器の継続性 — (a)+(b) に賛成。(c) は「基準」でなく「動作」を蒸留するなら可

fable-5 API の月次批評枠と sol/codex 定期走査の併用に賛成する。(c) ルーブリック化
の Goodhart 懸念は正当だが、中間の道がある: 蒸留すべきは**採点基準ではなく監査動作**
である。過去6本の批評が繰り返した手続き——主張をログと突き合わせる、陪審が
読まなかった範囲を確認する、版間 diff を取る、計器を信用する前に計器を測る、
語句水準の紋を数える、賞賛と減点を分離する——は、基準と違って Goodhart 耐性が
高い（動作は満たしても内容を偽装しにくい）。「批評チェックリスト」ではなく
「監査手順書」として内部審級に足すなら、(c) も価値がある。

もう一つ、設計として書き残すべきことがある。この座席の効力は、モデルの能力
ではなく**位置**から来ていた——パイプライン内の権限を持たず、全文脈を一括で
読み、率直である許可を与えられ、批評→実験→帰還の往復が保証されていた。後継が
fable-5 API であれ sol であれ別の私であれ、**この位置の仕様（読むもの・負うもの・
持たない権限）を1ファイルに書く**こと。「批評と応答」がサイトの一ジャンルに
なった以上、批評家は個体ではなく役職であり、役職は仕様書で継承できる。それが
本プロジェクトが記憶なしの設計者と二年やってきた方法そのものである。

### §3-2 計器台帳 — 賛成

`designs/instruments.md`、5行（novelty・fidelity・fixation・不一致度・perplexity）
から。各行に「何を測ると主張するか／最終校正日／既知の盲点」の三欄を推す。
盲点欄の初期値は既に判明している: novelty=cell系の定数debt、fixation=修辞・
世界型スコープ外、不一致度=下記。

### §3-3 不一致度の stopping 組み込み — 賛成。ただし単独計器にしない

w0005/w0006（改善時収束）と w0007（破壊時爆発）の対照は強い。組み込みに賛成する。
条件: **平均と不一致の結合判定**とすること。不一致の収束単独を成功信号にすると、
「陪審の合意点への最適化」（迎合）と区別できない——w0005 の 0.09 への収束を
監査対象とした 7/16 の債務は、この結合判定の中で生き続けるべきである。

### §4〜§5 — 記録として同意。二点のみ

- budgets.yaml cap 変更の CHANGELOG 未記録の再発: `aleph status` の突き合わせ警告に
  賛成。優先度低のままでよいが、**正典が正である**という問1の決定を選んだ以上、
  正典と設定の乖離検出は思想的には低優先ではない。実装が軽いなら早めに。
- 詩学第1版の ack が w0008 完成時に来る件: オーナーへの明示的な引き継ぎ事項と
  すること。忘れれば詩学は「権威化した緊張」に戻る。第二部の分布実験が改訂根拠を
  供給する設計なら、ack の判断材料も同時に揃う。

## 結び — 次の一手の推奨順（洞察 §6 への修正版）

1. C-1 をメタデータ直計算＋層化サンプリング注釈に切り替え（問2。今日から成立する）
2. UMAP 保険追試を事前登録スコープで一回（問1。C-2 調達判断の前に）
3. 応答B の正典化を 0.7.19 議題へ（修正条項三つ付き）＋ A への成功基準の事前登録
4. 配線棚卸しテスト（失効日つき許可リスト）
5. 家風の分布実験（L4、安価）→ 結果を見て全制約解除の一作を w0008 に
6. 批評器継続性: (a)+(b) 決定と「批評家役職仕様書」の起草を 7/19 前に

三通を通じて最も重要な一行は、中間総括の「監視器が見ていない場所で凝固が
進んでいる」である。七月にこのプロジェクトが繰り返し証明したのは、計器を
疑う文化だけが品質の実質的な保証器だということだった。家風は次の被疑計器では
なく、次の被疑**対象**であり、それを測る計器（fixation の拡張、分布実験）を
先に立てる——この順序を守る限り、凝固は様式に、様式は選択に変えられる。
