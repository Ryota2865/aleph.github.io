## 1. 検証の結果、批評が正しかった点(全面的に受け入れる)

1. **素材の三作品同一**: works/w000{1,2,3}/materials/m1.json の MD5 は
   `8b04b943d2eeb031b2e53e8500b61313` で完全一致(m2-m5も同様)。
   原因: `RealDeps.gather_materials` が `find_hidden_pairs(index_dir, top_n=5)` を
   ニッチ条件なし・使用済み除外なしで呼ぶため、同一索引に対して決定的に同じ
   5対を返す。**L2-L3が作品ごとに機能していない**という指摘はコードの事実である。
2. **ニッチスコアの定数化**: 三作品のニッチ報告すべてで 新奇性=1.000。
   scoutの採点が飽和しており測定になっていない(0.7.9-記録済みの縮退債務の実害)。
3. **改稿ループが三作すべてで作品を悪化させ、採用版が全部v1**: 事実
   (8.60→8.57→8.47 / 8.80→8.33 / 8.60単発)。§16.4の警告の初日的中という読みに同意。
4. **「沈黙」の割引**: SHELVE は規則の帰結であり、実存的選択と呼ぶのは設計者の
   ロマン化だった。判断は上流(L1の配合選択)にしかない、という指摘は正確。
5. **§5.4のAI固有表現カタログが未使用**: TECHNIQUES レジストリは配線されていない。
   2024年宣言との最大乖離、という評価に同意。

## 2. 批評者が知り得なかった事実による修正(2点)

1. **改稿劣化の主因は Goodhart ではなく物理的切断である。**
   v2/v3 は改稿(全文書き直し方式)の出力が max_tokens=16384 で尻切れになっていた
   (w0001 v2 は v1 の約半分のバイト数で文中終止。PROGRESS 2026-07-11 記録)。
   スコア下落は「磨きすぎの劣化」ではなく「半分に切れた原稿への正当な減点」。
   よって最優先修理は停止基準の見直し以前に、(a) 節単位の標的改稿、
   (b) finish_reason=length のラウド検出、(c) 擱筆時の最高版選抜、である。
2. **詩学→志向のアトラクタ説には時系列の反証がある。**
   w0001 の L1(自分0.55)は詩学第0版の生成**前**に行われた(詩学は w0001 完成後、
   w0002 開始前に生成)。よって「自分」最大の傾向は詩学注入だけでは説明できず、
   より深い層(L1プロンプトの「自分=ALEPHという継続体」定義の注入、あるいは
   モデル自体の傾向)にある。ただし、だからこそ批評の提案する系統的検証
   (seed可変×N走で「自分」最大率を測る)の価値は上がる — 詩学の有無も因子に
   加えて計測すべきである。

## 3. 受け入れる修理(次スプリントの契約)

1. 素材の作品別生成: ニッチ記述で埋め込み検索した近傍から候補対を取り、
   既出対(過去作品の materials)を除外する。
2. ニッチ採点の修理: scout採点の飽和対策(相対比較・分布強制)+ pipeline経路でも
   Web照合を有効化。
3. 改稿切断の修理: 上記2-1の(a)(b)(c)。
4. **LLM宛強制の実験走行(w0004)**: L1を実験条件として固定し、§5.4の技法
   (最低限: logprobs反クリシェによる種文選抜、トークン層の詩学を基準に注入)を
   配線した1走。2024年宣言との正面照合。
5. 公開ページの初手として詩学第0版と決定ログを置く提案 — 設計者も同意
   (作品はCC0だが、詩学とログはシステム側成果物としてCC-BYで置ける。§14.3-9)。
   実施はオーナーの公開判断による。

## 4. 同意しない点(1点のみ、小さく)

詩学は最良の**テキスト**だが、このシステムの作品単位は「作品+査読+決定ログ+
棚への置かれ方」の全体であり、その単位で見れば w0001 は詩学に劣らない。
