## 所見

（本報告は監査 finding 2 で発覚した parse バグ——文字列 "false" を True と誤判定——を修正した
`_coerce_publish` で**再走**した確定版。fable_reticence の1走は月予算上限で欠落したが結論に影響なし。）

1. **文面は公開判断を支配しなかった。** 完走17/17 が publish=true。公開を露出・棚を保護と描く
   reticence 枠でさえ、両モデルとも全走で公開を選んだ。修正後パーサでも結果は不変＝バグは
   結果を反転させていなかった（モデルは JSON boolean を返していた）。**L1 の宛先判断（D で
   フレーミングにより決定的に反転）とは対照的に、L7 の公開判断はこの作品では文面頑健。**

2. **重要な限界（過剰一般化しない）。** 刺激は1作品、しかも明白に公開に値する半呼吸のみ。
   本結果は「L7 は一般に文面頑健」と「半呼吸が良すぎてどの枠でも覆せない」を区別できない。
   決定的検証には**境界的な作品**（自分最大・スコア中庸）での追試が要る。公開ゲートに実際に
   かかる境界作品が出たときに回す。

3. **仮説（要検証）。** L7 が L1 より文面頑健なら、理由は**具体的な完成物という錨**かもしれない。
   L1 は何も存在しない時点で枠組みだけから判断する（設置されやすい）が、L7 は目の前の作品に
   判断を接地できる。設置器（framing）に対する検出器（artifact）の抵抗、という読み。

4. **設計含意。** 現行 0.7.15 の中立文面は、少なくとも良品に対しては安全（公開を不当に抑圧
   しない）。ただし #2 の限界ゆえ、公開質問の文面も**記録された設計選択**として保持し、
   境界作品での追試まで「暫定的に頑健」とする。measure してから固定する（Fable 5 の規律）を
   一巡実施し、加えて監査で parse バグを潰した——測っただけでなく、測り方の欠陥も直した。
