## 1. 結論は証拠に対して過大か

戦略判断としての「コーパス拡張へ進む」は支持します。  
ただし、方法論上は「クラスタリング側の問題ではない」と断定するには過大です。

より正確にはこうです。

> 現行の `PCA(64) -> sklearn HDBSCAN` 系では、`min_cluster_size` を 10〜40 で振っても、被覆測定に使える細かい意味クラスタは出なかった。

これは十分に言える。一方で、`min_samples`、`leaf`、PCA次元、UMAP系縮約、cluster_selection_epsilon まで含めた「クラスタリング側全体」を棄却したわけではありません。

ただし重要な補足があります。実データを読むと、`embeddings.npy` のノルムは全件ほぼ `1.0` でした。つまり実装は明示正規化していませんが、保存済み bge-m3 ベクトルはすでに単位ノルムです。したがって「生ベクトルのノルム差で euclidean が壊れている」という疑いは、この索引では弱いです。

また sklearn HDBSCAN の既定では `min_samples=None` が実質 `min_cluster_size` と連動します。今回のスパイクは「min_cluster_size だけ」ではなく、既定設定上は `min_samples` も一緒に動いています。これは既定経路の検証としては妥当ですが、`min_samples` を小さく固定した場合の検証にはなっていません。

## 2. 「巨大クラスタ + 大量ノイズ」の診断

これは単独では **(a) コーパスの真の均質性** の署名とは読めません。より妥当なのは、次の複合です。

- **(c) 密度ベース手法と連続的な密度勾配の相性** が主因候補
- **(b) 高次元埋め込み・PCA縮約による局所構造の鈍化** が副因候補
- **(a) 青空文庫の時代・文体・言語圏の狭さ** は、クラスタの中身の偏りを説明する上位要因

読んだ現ファイルでは、95,690 チャンク中、クラスタ化されたのは 9,058、ノイズは 86,632 でした。提示文の 85,663 とは少しずれますが、診断は変わりません。label 0 は全チャンクの約9%で、クラスタ化された点の約95%です。

安価な追加測定として有効なのは以下です。

- raw cosine k-NN 距離分布を、クラスタ点とノイズ点で比較する
- k-distance curve を `k=5/15/40` で描き、明確な膝があるか見る
- `min_cluster_size` と `min_samples` を分離し、`min_samples=5/10` 固定で再実行する
- `cluster_selection_method="leaf"` を1回だけ試す
- PCA次元を `10/15/64` で比較する
- ノイズ点を層化サンプリングし、era/form/theme を注釈する

今回こちらで読んだ範囲では、raw cosine kNN でもクラスタ点はノイズ点より近傍が明確に近い。つまり「完全に距離が壊れている」というより、「濃い局所ポケットだけが拾われ、広い文学的連続体がノイズに落ちている」と読むのが自然です。

## 3. 機会費用の判断

私は **直ちにコーパス拡張へ進む** 方を推します。

理由は、追加スパイクで 10〜30 クラスタが出ても、それは青空文庫内の分解能改善であって、era・言語圏・非文学形式の欠落は直りません。ALEPH の目的が「探索可能な軸を増やす」ことなら、HDBSCAN の粒度改善より、二次コーパス・Gutenberg/Wikisource 選抜の方が期待値が高いです。

ただし「以後この軸へ戻らない」は強すぎます。正しくは「コーパス拡張をブロックする課題ではない。クラスタ単位の被覆測定を再び意思決定に使う段階で、軽量な sanity check として戻る」です。

## 4. 被覆測定 C-1 への含意

4クラスタ単位の C-1 は、**コーパス被覆測定としては成立が弱い**です。  
成立しているのは「現行 HDBSCAN が検出した高密度モードの注釈レポート」です。

問題は3つあります。

- ノイズが約90%なので、コーパスの大半が被覆計算から落ちている
- 巨大クラスタの代表3チャンクが、その8,583チャンク全体を代表する保証がない
- 4クラスタから作る `4/256` は、統計量というより分類器と粒度の副産物

したがって、C-1 はクラスタ単位ではなく、**作品単位の層化サンプリング + 作品内チャンク注釈**へ切り替えるべきです。クラスタは層化の補助情報として使う程度がよいです。特にノイズ点を必ず含めるべきです。

**署名**: Codex (GPT-5) / 推論強度: 高

**推奨する次の一手**

1. S-1/S-2 の二次コーパス小規模 pilot へ進む。HDBSCAN追加探索でブロックしない。
2. C-1 は「クラスタ被覆」から「作品/チャンク層化サンプル注釈」へ改める。
3. クラスタリング再検証は、`min_samples` 分離・PCA次元・leaf の軽量チェックとして後で戻す。
