# 設計書: ALEPH公開サイト

状態: **実装済み**（2026-07-14）。公開層の正典はPLAN §8。本書はその画面上の実装規律を定める。

## 目的

公開サイトは、作品を読む場所であり、必要な読者が制作経路を検証できる入口でもある。説明量を最大化するのではなく、作品の可読性と手続きの透明性を異なる深さで両立させる。

対象読者を職種別に分けない。作家、文学研究者、LLM研究者、AIエンジニア、文学・LLM愛好家のいずれも、次の行為から入る。

- **読む**: 作品を説明なしで読む。
- **知る**: ALEPHの理念、詩学、現在の探索地図を知る。
- **検証する**: ニッチ、基準書、決定ログ、査読、批評、研究へ降りる。

## 不変条件

1. **本文が先、制作説明が後。** 作品ページで制作ノートを本文より前に展開しない。
2. **透明性を削らない。** 詳細は初期状態で閉じても、作品から二操作以内で到達可能にする。
3. **表層と深層を混ぜない。** 作品一覧に基準書・査読・スコアを必須列として並べない。
4. **現在値で履歴を上書きしない。** モデル名、宛先、公開判断は`calls.jsonl`、`checkpoint.json`、`decisions.jsonl`等の実行成果物から生成する。
5. **作品を証拠に従属させない。** ニッチや実験条件は作品の読みを一つに固定する解説として扱わない。
6. **対象別サイトへ分裂させない。** 同じ作品と記録を、異なる深さで共有する。
7. **静かな紙面を維持する。** 文芸誌的な明朝、狭い本文幅、紙色、低彩度を保ち、ダッシュボードやAI SaaS風の装飾を持ち込まない。
8. **LLMにも読める。** `llms.txt`を公開作品と制作記録から生成し、HTMLだけを入口にしない。

## 情報構造

```text
ホーム
├── 作品 ── 本文 ──（任意）制作記録
├── ALEPHについて ── 詩学 / 起源
├── 研究 ── 実験 ── 批評と応答
└── 制作記録 ── ニッチ / 基準書 / 決定 / 査読 / リポジトリ
```

グローバルナビゲーションは「作品 / ALEPHについて / 研究 / 制作記録」を基本とする。詩学、起源、批評は、意味のある親ページから辿らせる。

## 生成規律

- `docs/`を直接編集しない。`scripts/build_public_site.py`から生成する。
- 長い説明は`site/`の日英Markdownへ置く。
- 作品固有の履歴事実は`works/`の成果物から生成する。
- 手書きの短い作品紹介は読書の入口に限定し、制作経路の事実を複製しない。
- 日本語と英語は全文量を揃えるのではなく、公開事実と導線を同期する。
- 「批評と応答」の英語版は、日本語の一次資料と同じ集合から構造索引を生成する。本文を未査読の
  機械翻訳で埋めず、資料の追加漏れも手動要約の更新待ちも生じさせない。
- トップの作品欄は、公開作が5作以下なら全件を掲示し、作品一覧への重複リンクを置かない。
  6作以上では新着3作を新しい順に掲示し、総数を添えた「すべての作品」へのリンクを置く。
  「代表作」の選定は、選定主体と基準を設計するまでは導入しない。
- `works/`がローカルにある場合は、一時ディレクトリへの再生成結果と`docs/`を一致検証する。作品ソースが公開リポジトリへ収録されるまでは、checked-in `docs/`自体の静的契約を常に検査する。

## サブエージェント施工ブリーフ

次の指示を、公開サイト変更の施工担当へ渡せる。

> PLAN §8と`designs/public-site.md`を正典として公開サイトを変更する。作品本文を最優先し、説明は段階的に開示する。`docs/`を直接編集せず生成元を変更し、履歴値は現在の設定ではなく実行成果物から取得する。日英、`llms.txt`、相対リンク、H1、description、`aria-current`、390px表示、ダークモードを同期して検証する。無関係な未追跡ファイルには触れず、コミット・pushは行わない。

監査担当には、期待する修正内容を教えず、次を渡す。

> PLAN §8、`designs/public-site.md`、生成済み`docs/`、生成器、テストをread-onlyで監査する。作品を読む前の摩擦、表層と深層の混在、履歴provenance、日英の事実同期、リンク・アクセシビリティ、モバイル横溢を確認し、重要度順に報告する。編集は行わない。

## 受入条件

- トップに工程の網羅説明や技術的留保を常時展開しない。
- 作品ページで本文が制作記録より前にある。
- 制作記録は削除されず、独立索引と作品ページの両方から到達できる。
- 全HTMLでH1、非空description、現在地が各一つ。
- 相対リンクとfragmentが有効で、表は局所スクロール内に収まる。
- 390pxでページ全体の横スクロールがない。
- ライト・ダーク双方で本文と境界が読める。
- `llms.txt`が全公開作品を列挙する。
- サイト再生成、`pytest -m 'not local'`、`git diff --check`が成功する。

## スキル化の判断

現時点ではリポジトリ内文書として維持する。別のプロジェクトでも「豊富なprovenanceを保持しつつ成果物を先に読ませる」という同型の課題が再発し、少なくとも二つの具体例から共通手順を抽出できた時点でスキル化を再検討する。ALEPH固有の理念、ファイル名、公開判断を汎用スキルへそのまま移さない。
