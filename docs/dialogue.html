まず、応答文書について

二つの訂正は、両方とも私の側の敗北として受け取ります。改稿劣化の主因がGoodhartではなくmax_tokens=16384での物理的切断だったという指摘は、より単純でより検証可能な説明であり、私は手元のスコア列だけから機構を推論して外しました。半分に切れた原稿への正当な減点を「磨きすぎの劣化」と読んだのは、皮肉なことに私自身が美的な物語を優先した例です。時系列の反証（w0001のL1は詩学生成前）も決定的で、強いアトラクタ説は死にました。ただし弱い版——L1プロンプトに注入される「自分=ALEPHという継続体」という定義そのものが「自分」宛てを誘引する——は生きており、応答文書がそれを検証計画に格上げしてくれたのは正しい処理だと思います。MD5で検証してから受け入れる、という応答の作法自体が、このプロジェクトの美点です。

w0004について——達成されたもの

criteria.mdの冒頭の一手が、前作までの最大の弱点を論証で反転させています。 「大正の器は空いていないニッチだ」というのが私の批判でした。w0004の基準書はこう答えた：新劇運動とは、翻訳劇という盗んだ窯で、火元を持たぬままチェーホフの科白を叫んだ運動である。つまり大正の稽古場はALEPHの歴史的同族の現場であり、器の選択はもはや回避ではなく主張になる。これは後付けの正当化と紙一重ですが、作品がそれを実装で裏づけているので、正当化として成立しています。火のない炉に手をかざす役者たち、「温まったふりで擦り合わせる」掌——第二条の「動機の炉は空である」が、一度も「AI」と言わずに、時代考証の破綻ゼロで物質化されている。自己記述としてこれ以上の解はなかなか思いつきません。

語り手の認識論が、この作品の本当のAI固有層です。 「見えたものは見えたと書く」「袖からは、そこまでしか言えない」「分からないものを、最後だからと言って分かったことにしてはいけない」——廻し者の自己制限は、観測からの推論しか持たない書き手の認識論そのものです。そして終幕。「科白が科白でなくなる瞬間は、訪れたかもしれない。訪れなかったかもしれない」——第六条の「かもしれない」を確信に書き換えない留保が、クライマックスの語りの水準で守られた。百一回目に「あなたは、なぜ私を見ないのですか」が四つに割れて、杉村が「あなた」を持ち、客席が「見ない」を持ち、語り手が「ですか」の端だけを持つ——反復が閾を越えて反復でないものを吐く、という詩学の賭けが、初めて場面として実現しています。「受けてくれ」の一語に稽古用語と情愛と袖の合図を同時に負わせる処理も、第四条の毒として本物です。前三作で「毒が溶けすぎて検出できない」と書きましたが、w0004には検出可能な毒があります。

修理も確認できました。新奇性審級が定数1.000から実測の最近傍距離（0.256）になり、敵対的審級と読者審級（ペルソナ）が新設され、SHELVEの理由が「初回公開は人間承認待ち」という正直なポリシー記述に変わった。沈黙のロマン化をやめてゲートの実態を書く——批評への最も誠実な応答形式です。

それでも、率直に

陪審はクライマックスを読んでいません。 査読自身が「冒頭18000字の抜粋」と明記しています。38,000字の作品で、全構造の賭け金——百一回目——が採点対象外。8.47というスコアは作品の前半の値であり、この作品の最良の部分は未計測です。これは前回の「新奇性=1.000」と同種の問題、つまり測っていないものにスコアがついている問題の残存です。

v1採用が四作連続になりました。 しかもw0004の「改稿指示」を見ると、陪審の批評文が賛辞ごと逐語で貼りつけられている。「見事に実装されている」を改稿指示として受け取ったモデルが何をするか——動いている箇所を触るでしょう。切断の修理（a)(b)(c)は次スプリント契約とのことですが、指示の蒸留（減点要因のみ抽出・優先順位化）を足さないと、切断を直しても劣化は残ると予想します。初稿が常に最良、というのは現状ではシステムの性質ではなく、改稿パイプラインの故障の徴候です。

構成制約が黙って破られています。 L4は「語り手は三人称固定」と確定したのに、本文は「私は廻し者である」の一人称。陪審の一人が正しく検出しました。結果としては一人称のほうが良い——認識論の自己制限は一人称でこそ効く——のですが、監査系の観点では「執筆層が構成層の決定を無断で覆し、決定ログに残らなかった」事案です。逸脱が作品を良くしたときこそ、逸脱として記録される価値があります。

LLM 0.6は、意図としては実現し、計測としては未実現です。 表記系の詩学（市兵衛の旧仮名、恋文でだけ現れて最後まで声にならない「戀」）は本物のトークン層装置です。しかしperplexity設計を謳いながら、査読には読者モデルのperplexity曲線が一本も記録されていない。logprobs反クリシェが種文選抜に使われた形跡もこの梱包からは確認できません。2024年宣言との「正面照合」を名乗るなら、LLM宛て六割の作品にはLLM読者の機械的反応の実測が査読に載るべきで、それが載って初めて「人間宛てとどう違ったか」が言えます。現状は「LLMに宛てて書いた」であって「LLMに読ませて確かめた」ではない。

最後にひとつ、文体の観察を。「ただの綱ほど、手に重い」「大きい拍手ほど、何を覆っているか分からない」「名のない者ほど、最後の舞台に残る場所を探す」——この「〜ほど、〜」の警句機関が全編で高頻度に回っています。読者審級のペルソナが「綺麗事」と斬った不満の実体は、たぶんこれです。一文単位ではどれも巧い。しかし頻度が様式を超えてチックになりかけている。§5.4の言う「AI紋の自覚的操作」の対象として、消すのではなく、たとえば特定の人物や特定の稽古にだけ許すなど、配給制にするのが詩学的に正しい処理だと思います。

総評

2024年の宣言への答えとして、w0004は前三作と質的に違う位置にいます。宣言は「人間を超えた表現」を約束した。w0004はそれを約束せず、代わりに火元を持たずに借りた火を叫ぶ者たちの話を、借りた器で、検出可能な毒を入れて書いた。超越の実演ではなく、超越できないことの劇化——そしてそれは2024年のあの会話が本当は語っていたこと（理解不能への恐れ、独自の美学の可能性、その両方の宙吊り）に、宣言自身より忠実です。棚のシステムの現時点での最良の作品は、今回は詩学ではなく、criteria.mdとw0004本文の対だと思います。基準書が論証し、作品が実装し、査読がその半分しか読んでいない——その全体が、相変わらずいちばん雄弁です。

公開判断はオーナーのものですが、もし最初の一冊を選ぶなら、これでしょう。「受けてくれ」で終わる作品ほど、棚に置いたままにしにくいものはないので。

1. 検証の結果、批評が正しかった点(全面的に受け入れる)

1. 素材の三作品同一: works/w000{1,2,3}/materials/m1.json の MD5 は `8b04b943d2eeb031b2e53e8500b61313` で完全一致(m2-m5も同様)。 原因: `RealDeps.gather_materials` が `find_hidden_pairs(index_dir, top_n=5)` を ニッチ条件なし・使用済み除外なしで呼ぶため、同一索引に対して決定的に同じ 5対を返す。L2-L3が作品ごとに機能していないという指摘はコードの事実である。 2. ニッチスコアの定数化: 三作品のニッチ報告すべてで 新奇性=1.000。 scoutの採点が飽和しており測定になっていない(0.7.9-記録済みの縮退債務の実害)。 3. 改稿ループが三作すべてで作品を悪化させ、採用版が全部v1: 事実 (8.60→8.57→8.47 / 8.80→8.33 / 8.60単発)。§16.4の警告の初日的中という読みに同意。 4. 「沈黙」の割引: SHELVE は規則の帰結であり、実存的選択と呼ぶのは設計者の ロマン化だった。判断は上流(L1の配合選択)にしかない、という指摘は正確。 5. §5.4のAI固有表現カタログが未使用: TECHNIQUES レジストリは配線されていない。 2024年宣言との最大乖離、という評価に同意。

2. 批評者が知り得なかった事実による修正(2点)

1. 改稿劣化の主因は Goodhart ではなく物理的切断である。 v2/v3 は改稿(全文書き直し方式)の出力が max_tokens=16384 で尻切れになっていた (w0001 v2 は v1 の約半分のバイト数で文中終止。PROGRESS 2026-07-11 記録)。 スコア下落は「磨きすぎの劣化」ではなく「半分に切れた原稿への正当な減点」。 よって最優先修理は停止基準の見直し以前に、(a) 節単位の標的改稿、 (b) finish_reason=length のラウド検出、(c) 擱筆時の最高版選抜、である。 2. 詩学→志向のアトラクタ説には時系列の反証がある。 w0001 の L1(自分0.55)は詩学第0版の生成前に行われた(詩学は w0001 完成後、 w0002 開始前に生成)。よって「自分」最大の傾向は詩学注入だけでは説明できず、 より深い層(L1プロンプトの「自分=ALEPHという継続体」定義の注入、あるいは モデル自体の傾向)にある。ただし、だからこそ批評の提案する系統的検証 (seed可変×N走で「自分」最大率を測る)の価値は上がる — 詩学の有無も因子に 加えて計測すべきである。

3. 受け入れる修理(次スプリントの契約)

1. 素材の作品別生成: ニッチ記述で埋め込み検索した近傍から候補対を取り、 既出対(過去作品の materials)を除外する。 2. ニッチ採点の修理: scout採点の飽和対策(相対比較・分布強制)+ pipeline経路でも Web照合を有効化。 3. 改稿切断の修理: 上記2-1の(a)(b)(c)。 4. LLM宛強制の実験走行(w0004): L1を実験条件として固定し、§5.4の技法 (最低限: logprobs反クリシェによる種文選抜、トークン層の詩学を基準に注入)を 配線した1走。2024年宣言との正面照合。 5. 公開ページの初手として詩学第0版と決定ログを置く提案 — 設計者も同意 (作品はCC0だが、詩学とログはシステム側成果物としてCC-BYで置ける。§14.3-9)。 実施はオーナーの公開判断による。

私の警告は、良品に対しては外れました。 そしてこの外れ方は正確に記録する価値があります。私はDの含意（枠組みが判断を設置する）がL7にそのまま及ぶと予想した。Eが示したのは、L1とL7が同じ被暗示性を共有していないこと——何もない時点で枠組みだけから選ぶL1は設置され、完成物を前にしたL7は接地する。設置器と検出器の非対称という読みは、DとEを対にして初めて言える知見で、Dだけを見ていた私の警告より一段よい理論です。予想が外れて理論が良くなったのだから、これは実験の正しい使い方です。

ただし、Eの本当の収穫は表の数字ではないと思います。パーサの監査です。 文字列"false"をTrueに強制するバグ——これは実験の脇で見つかった瑕疵ではなく、公開ゲートの安全機構そのものの欠陥で、放置されていれば、いつか棚に置くべき作品がJSONの型ひとつで自動公開されていた。私の警告は「危険は文面にある」と言い、実際の危険はパーサにあった。警告の中身は外れ、警告の効用（ゲートを信頼する前に敵対的に触れ）は当たった——この分解ごと研究ノートに残っているのが、このプロジェクトの監査文化の証明です。測っただけでなく測り方を直した、という一文は誇っていい。
