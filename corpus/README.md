## corpus/aozora/works.jsonl — 青空文庫（パブリックドメイン）

- **取得元**: Hugging Face `globis-university/aozorabunko-clean`
  （https://huggingface.co/datasets/globis-university/aozorabunko-clean）。
  青空文庫全作品をルビ記法・入力者注（`《...》`・`[#...]`）除去済みでクリーニングした
  データセット。データセット自体のライセンスは **CC BY 4.0**（globis-university）。
  収録テキストの原文は青空文庫の**パブリックドメイン作品**（著作権法上、保護期間満了）。
- **取得日**: 2026-07-09
- **取得方法**: `scripts/build_aozora_corpus.py`
  （`uv run --with datasets python scripts/build_aozora_corpus.py`）。
  `datasets.load_dataset()` でHFからロードし、下記PD判定でフィルタしてJSONL化。
- **件数**: 16,950件（元データセット16,951件のうち、本文が空の1件のみ除外。
  重複IDなし）
- **総サイズ**: 約704 MiB

### PDフィルタ（判定根拠）

青空文庫のマスターメタデータには2種類の著作権フラグがある:

- `作品著作権フラグ`（作品側。訳者・編者等の権利による制限の有無）
- `人物著作権フラグ`（人物側。著者・訳者等それぞれの権利の有無）

**両方が「なし」（完全にパブリックドメイン）の行のみ採用**。
`globis-university/aozorabunko-clean` の16,951行は実測で全行すでに
両フラグ「なし」だった（データセット作成者が構築時に著作権保護作品を
除外済みと推定される）が、将来のデータセット改訂で保護作品が混入しても
検出できるよう、`build_aozora_corpus.py` はこのフィルタを明示的に実装して
適用している（信頼せず都度検証）。除外件数: 0件（PD以外を理由に除外された
行なし。本文空1件のみ別理由で除外）。

**追加のサニタリチェック**（2026年時点の日本の著作権法との整合性検証）:
死後70年ルールは2018年12月30日のTPP11整備法施行時点で**既にパブリック
ドメインになっていた作品には遡及適用されない**（旧50年ルールのまま）。
したがって、なお保護期間中のはずの「1968年以降に没した著者」の作品が
本コーパスに紛れていれば異常のシグナルとなる。実測した死没年の最大値は
**1967年**（死後50年ルールで2017年末にPD化 → 2018年の延長改正前なので
遡及保護されない）であり、1968年以降没の著者は**0件**。PD判定が実際の
著作権法の経過措置と整合していることを確認した。

### スキーマ

```json
{
  "id": "作品ID（青空文庫の作品ID、6桁ゼロ埋め文字列。ユニーク）",
  "title": "作品名",
  "author": "姓 名（半角スペース区切り）",
  "text": "本文全文（ルビ・入力者注は除去済み）",
  "meta": {
    "...": "青空文庫マスターメタデータの全フィールドをそのまま格納",
    "作品著作権フラグ": "なし",
    "人物著作権フラグ": "なし",
    "文字遣い種別": "新字新仮名 / 新字旧仮名 / 旧字旧仮名 / 旧字新仮名 / その他",
    "生年月日": "著者生年月日",
    "没年月日": "著者没年月日",
    "底本名1": "底本（第一）の書誌情報。底本出版社名1・底本初版発行年1等も同様",
    "footnote": "底本・入力者・校正者クレジット等の後書き原文（元データセットのfootnoteフィールド）"
  }
}
```

`meta` のキーは元データセットの `meta` フィールド（青空文庫マスターDBの
列名、日本語）をそのまま踏襲。タイムスタンプ型（公開日・最終更新日等）は
ISO 8601文字列に変換済み。

### 品質チェック

- 件数: 16,950件（要件の10,000件を超過）
- ランダム5件を目視検査（`scripts/qa_aozora_corpus.py`）: 実際の日本語文学
  テキストであること、文字化けがないこと、ルビ記法`《》`や入力者注`[#...]`が
  残っていないことを確認。1件のみ`《民話を描く》`のようなヒットがあったが、
  これは本文中の見出し的強調表現であり、ルビ・入力者注の残留ではないことを
  文脈確認済み。
- スキーマ違反0件、id重複0件（`scripts/qa_aozora_corpus.py`）。
